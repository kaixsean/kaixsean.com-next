不知道你有沒有發現，本部落格的右側區塊有一個 Switch to Brave 的按鈕?就在我用了 Brave Browser 三個月後，BAT 幣終於開始有所進帳。原來上網滑著滑著，真的有錢賺！！！  
不過，為什麼上網還能賺錢，這就要從 Brave Browser 的理念與概念說起了。

## 什麼是 Brave Browser ?
與多數瀏覽器相同，Brave 同樣基於 Chromium，與 Chrome 有相同的核心。Brave 由 Javascript 創始人及 Mozilla 前 CTO 、共同創始人，Brendan Eich， 於 2015 年創立。  
Brave 至今在多個主流平台都下載得到，並提供包含預設 DuckDuckGo 在內的五大主流搜尋引擎。

## 為何使用 Brave Browser ，它的目標是什麼？
Brave Browser  想解決的問題是，源於廣告及追蹤器而導致日益嚴重的隱私問題，並希望藉由給予網站主、創作者加密貨幣的方式，擺脫廣告及網頁追蹤器，以促進使用者的隱私，使得網站主及創作者可以不再依賴廣告提供免費內容。

## 咦？我上網本來就不用付費呀！
沒錯，一般來說，我們上網觀看的許多內容，確實不用付費。但那是因為已經有人幫我們付費了，也就是所謂的「羊毛出在狗身上，豬付費」。現今架設網站，頻寬、主機的水電等都是成本，因此廣告主付錢給網站主，讓網站主可以在網站中置放廣告，使得受眾（你）可以在不用付費的狀態下，免費取得網站內容觀看。所以，當我們上網時，許多內容本身並不是免費的，而是已經有人買單了!

## 這樣不是很好嗎？有啥問題？
確實，這樣真的很棒。「現代廣告」這樣的模式，從早期的報章雜誌、電視廣播，一直到現今的網際網路，人類已應用多年。透過廣告產業，養活數十億人，促進的經濟效應不計可數，可以說是人類世界的偉大發明之一。但科技進步，為了能夠更加精準抓對可能消費的對象，現今許多網站，都有內置一堆追蹤器，以獲得使用者的喜好。這就導致，我們常常可能只在服飾購物網站買了一件衣服，使用臉書時就推送給你一堆內褲廣告。是否無傷大雅，就見仁見智，對於注重隱私的人來說，肯定是感覺不太舒服吧。

因此，Brave Browser  希望透過一個金流機制來解決這樣的問題。利用發行 BAT (Back Attention Token，注意力幣) ，來代表你在一個網站上花費的注意力，透過兌換 BAT 的方式成為網站主的收入來源。促使網站主可專注於提供更好且吸引人的內容，而非貼滿滿滿的廣告。

那為了實現這個目標，因此 Brave Browser  設計了 Brave Reward 計劃。

## 什麼是 Brave Reward
<iframe src="https://player.vimeo.com/video/332258054" width="640" height="360" frameborder="0" allow="autoplay; fullscreen" allowfullscreen></iframe>
<p><a href="https://vimeo.com/332258054">Brave Rewards</a> from <a href="https://vimeo.com/user64461481">Brave</a> on <a href="https://vimeo.com">Vimeo</a>.</p>

Brave Reward 計畫是透過觀看瀏覽器的廣告以獲取 BAT，每一次廣告，都是取其利潤的 70% 轉換成 BAT 分配給使用者。再經由計算注意力，分配 BAT 給予網站主獎勵。使用者可以設定由系統自動配給，或是自行贊助。雖然還是有屬於瀏覽器的廣告，但以注重隱私為核心的 Brave Browser  宣稱，絕不會使用你的隱私，這部分就看使用者的信任度了。

## BAT 幣現在能在哪裡交易
BAT 是加密貨幣的一種，目前已經可以在多數的交易所交易。

## 如何加入 Brave Creator
前面有說到，Brave Browser 會自動分配 BAT 給網站主，但並不是所有網站都可以拿到，而是要到下面的網址，登記成為 Creator 創作者才可以。

[Earn more for content you publish to the web - Brave Creators](https://creators.brave.com/)

登記後，只要有使用 Brave Browser 使用者光臨你的網站，你就可以被分配到一定額度的 BAT。如此，透過結合 瀏覽器、使用者、網站主（或創作者）三方，變成一個 BAT 生態鏈、經濟圈。

## 結論
網路生態在近五年有明顯的變化，自從「劍橋分析」事件後，廣告與隱私的拉鋸日益激烈。但沒有廣告，要如何創造收入？特別是對於像媒體這樣的類型，無法如同商品那樣實體般交易，該如何在數十年來都免費觀看的習慣下，開創新的商業模式？區塊鏈似乎是個可以嘗試的解方。而利用區塊鏈的特性來解決這個問題的，除了 Brave Browser  外，還有讚賞公民 [Civic Liker - Liker.Land](https://liker.land/civic/) 。但兩者有許多的不同，Brave Browser   從瀏覽器的角色出發，隱私是其最主要考慮的議題。  
而我也觀察到，目前既有的趨勢是，一些 Blogger 除了放原本的 Google 廣告之外，也加入 Brave Creator。我想應該也是在觀察風向，畢竟目前 Brave Browser  的月活躍用戶在今年六月才剛達到 1500萬，市佔率離 Google Chrome 還有一大段距離。Brave Browser  能不能透過 BAT 的機制來解決隱私及網站主的收入問題，都還需要時間來證明。目前維基百科以及搜尋引擎 DuckDuckGo 都已經加入了 Brave Creator，也期待有更多網站的加入，才能擴大及觀察到這樣的生態圈是否可行。雖然使用 Brave Browser 還是會有透過瀏覽器的廣告，但基於 Brave Browser 是開放原始碼軟體，我覺得還是值得信任。  
至於在日常使用上，由於 Brave Browser 基於 Chromium ，與 Google Chrome 有相同的核心，所以相容性上幾乎沒有什麼問題，外掛也相通。唯一比較有問題的是，有時候因為廣告阻擋器的關係，導致有些網站在操作時會有問題，這時候把廣告阻擋器關掉即可。

而我自己，除了使用 Brave Browser 有參與 Brave Reward 之外，本部落格也有加入 Brave Creator，前面提到的讚賞公民也有加入，算是多方嘗試。除了觀察的同時，也加入這樣的生態圈，期能更貼身了解，或者也可以參與成長。

若有興趣的話，也一同加入嘗試看看吧！
